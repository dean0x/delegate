## Implementation Details

The fix added two new Maps to `TestEventBus`:

1. **subscriptionToHandler**: Maps subscription ID to `{eventType, handler}` - enables proper handler removal when `unsubscribe()` is called.

2. **handlerToSubscription**: Maps original handler function to subscription ID - enables `removeListener()` to find the correct subscription for handlers registered via `once()`.

Key changes:
- **waitFor()**: Captures subscription ID and unsubscribes in both timeout and success paths
- **once()**: Captures subscription ID, stores mapping, and unsubscribes after first invocation
- **removeListener()**: First checks handlerToSubscription Map, then falls back to direct removal
- **unsubscribe()**: Now actually removes the handler from the handlers Set
- **unsubscribeAll()**: Clears both new Maps
