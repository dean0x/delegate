## Architecture & Design

### Key Architectural Decisions

1. **Handler-Owned Graph**: DependencyHandler now owns the in-memory graph and performs all business logic (cycle detection, validation). Repository is pure data access.

2. **Incremental Updates**: Graph updates happen O(1) after each database operation, replacing O(N) cache rebuilds.

3. **Transaction Safety**: Graph updates occur AFTER successful database transactions, not before.

4. **Technical Safeguards Over Documentation**: `npm test` is blocked at the command level to prevent accidental crashes.

### Technology Stack
- **Runtime**: Node.js >= 20.0.0
- **Language**: TypeScript (ES Modules)
- **Database**: SQLite with WAL mode (better-sqlite3)
- **Testing**: Vitest 3.2.4
- **Protocol**: Model Context Protocol (MCP)

### Design Patterns in Use
- Event-Driven Architecture (EventBus)
- Result Pattern (no thrown errors in business logic)
- Dependency Injection
- Repository Pattern
- DAG (Directed Acyclic Graph) for dependencies
