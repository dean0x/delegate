## 🏗️ Architecture & Design

### Key Architectural Decisions

1. **Event-Driven Architecture with EventBus**
   - Why: Decouples components, enables extensibility, supports request/response patterns
   - Impact: EventBus constructor requires `(config, logger)` - major test update needed
   - Pattern: All operations (commands AND queries) go through EventBus

2. **Domain Model Immutability**
   - Why: Prevents accidental mutations, ensures data integrity
   - Impact: Tasks created with `Object.freeze()`, TaskFactory must create new objects not mutate
   - Pattern: All domain objects are frozen, updates return new objects

3. **Result Types for Error Handling**
   - Why: No exceptions in business logic, explicit error handling
   - Impact: All operations return `Result<T, E>` not throw
   - Pattern: `{ ok: true, value: T } | { ok: false, error: E }`

4. **Configuration Schema with Defaults**
   - Why: Security hardening (max 1h timeout, max 32 cores), experimental features opt-in
   - Impact: Schema grew from 5 to 20+ fields, all have defaults
   - Pattern: Zod schema with `.default()` for all fields

5. **Worktrees Experimental and OFF by Default**
   - Why: Complex feature, not needed by most users, opt-in safer
   - Impact: `useWorktree: false` by default, tests expecting `true` fail
   - Pattern: Experimental features default OFF with explicit opt-in
