## 🏗️ Architecture & Design

### Key Architectural Decisions

1. **Event-Driven Dependency Management**
   - Why: Decouples dependency logic from task lifecycle, enables extensibility
   - Impact: All dependency operations flow through EventBus (TaskDelegated, TaskCompleted, TaskUnblocked)
   - Pattern: Pure event-driven with no direct repository access outside handlers

2. **DAG Validation with Cycle Detection**
   - Why: Prevents deadlocks from circular dependencies (A→B→A or A→B→C→A)
   - Impact: Every dependency addition validates entire graph using DFS (O(V+E) complexity)
   - Implementation: `DependencyGraph.wouldCreateCycle()` using depth-first search

3. **TOCTOU Protection via Synchronous Transactions**
   - Why: Prevent race conditions where concurrent checks both pass but create cycle
   - Impact: Uses better-sqlite3's synchronous `.transaction()` for true atomicity (no event loop interleaving)
   - Security: Verified with concurrent A→B and B→A test

4. **Blocked Task Queuing**
   - Why: Tasks with unmet dependencies should wait, not execute
   - Impact: QueueHandler checks `isBlocked()` before enqueuing; TaskUnblocked event triggers delayed queueing
   - Pattern: Tasks remain in QUEUED status but outside queue until dependencies resolve

5. **Handler-Level Graph Caching**
   - Why: Repository-level cache was bypassed by repeated `findAll()` calls
   - Impact: Graph cached at handler level, invalidated on TaskDependencyAdded events
   - Performance: Reduces N+1 query pattern to O(1)

### Technology Stack
- **Language**: TypeScript 5.x
- **Runtime**: Node.js 20.0.0+
- **Data Storage**: SQLite3 with better-sqlite3 driver (WAL mode, synchronous transactions)
- **MCP**: Model Context Protocol SDK for Claude integration
- **Testing**: Vitest with 51/51 tests passing (100% success rate)
- **Key Dependencies**: better-sqlite3 (ACID transactions), @modelcontextprotocol/sdk, zod (validation)

### Design Patterns in Use

- **Result Pattern**: All functions return `Result<T>` instead of throwing exceptions (no try/catch in business logic)
- **Event-Driven Architecture**: EventBus coordinates all components (TaskManager, DependencyHandler, QueueHandler, WorkerHandler)
- **Dependency Injection**: All components receive dependencies via constructor (testability, no global state)
- **Repository Pattern**: Data access abstracted behind interfaces (TaskRepository, DependencyRepository, OutputRepository)
- **Immutability**: All Task and Dependency fields are `readonly` (return new objects on updates)
- **Factory Pattern**: `createTask()` factory ensures consistent task creation
- **Directed Acyclic Graph (DAG)**: Dependency relationships form DAG validated on every addition
