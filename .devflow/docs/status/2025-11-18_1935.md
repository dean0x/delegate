## 🏗️ Architecture & Design

### Key Architectural Decisions

1. **Event-Driven Dependency Management**
   - All dependency operations go through `DependencyHandler`
   - Emits `TaskDependencyAdded` events for compatibility
   - Cache invalidation via event bus

2. **Repository-Level Graph Caching**
   - In-memory `DependencyGraph` cached at repository layer
   - Invalidated on any dependency mutation
   - Prevents N+1 query pattern for cycle detection

3. **Synchronous Transaction Pattern**
   - All validation and insertion within single `db.transaction()`
   - No `await` inside transaction (prevents event loop yielding)
   - TOCTOU protection: cycle detection at insert time

4. **Result Pattern for Error Handling**
   - All repository methods return `Result<T, Error>`
   - Exceptions only for unexpected system errors
   - Consistent error handling across all layers

5. **DFS with Memoization for Depth Calculation**
   - Recursive depth-first search with memoization
   - O(V+E) complexity handles diamond graphs efficiently
   - Function-scoped memo (no memory leak)

### Design Patterns in Use

1. **Repository Pattern**
   - Clean separation: domain logic vs. database access
   - All data access through repository interfaces
   - Prepared statements for SQL injection prevention

2. **Dependency Injection**
   - All dependencies injected via constructor
   - Enables easy testing with mocks
   - Clear dependency graph

3. **Result Pattern**
   - Explicit error handling without exceptions
   - `ok(value)` for success, `err(error)` for failure
   - Pattern matching for error handling

4. **Event-Driven Architecture**
   - EventBus coordinates all components
   - Handlers specialized by concern (persistence, queue, dependency)
   - Zero direct state management
