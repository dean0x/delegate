## 🏗️ Architecture & Design

### Key Architectural Decisions

1. **Event-Driven Architecture with EventBus**
   - Why: Provides loose coupling between components, enables testing, supports extensibility
   - Impact: All service operations go through EventBus (both commands and queries)
   - Pattern: Request-response pattern for queries, fire-and-forget for events
   - Key files: `src/core/events/event-bus.ts`, `src/services/handlers/*`

2. **Result Type Pattern (No Exceptions in Business Logic)**
   - Why: Makes error handling explicit and type-safe
   - Impact: All business logic returns `Result<T, E>` instead of throwing
   - Pattern: Railway-oriented programming
   - Key files: `src/core/result.ts`

3. **Dependency Injection via Container**
   - Why: Makes testing trivial, promotes SOLID principles
   - Impact: All services instantiated through DI container
   - Pattern: Constructor injection with factory functions
   - Key files: `src/bootstrap.ts`, `src/core/container.ts`

4. **Heap-Based Priority Queue (v0.2.3)**
   - Why: O(n²) array operations were bottleneck for large task queues
   - Impact: 100x performance improvement (1000 tasks: 1000ms → 10ms)
   - Pattern: Min-heap with Map-based indexing for O(1) lookups
   - Key files: `src/implementations/task-queue.ts`
