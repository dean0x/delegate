## 🏗️ Architecture & Design

### Key Architectural Decisions
1. **Centralized Configuration with Zod Validation**
   - Why: Type safety, runtime validation, single source of truth
   - Impact: Eliminates scattered constants, enables user configurability, prevents invalid configs

2. **Dependency Injection with Configuration-First Approach**
   - Why: Testability, maintainability, proper inversion of control
   - Impact: All components now receive Configuration in constructor, bootstrap.ts manages all DI

3. **Environment Variable Support with Consistent Naming**
   - Why: Standard deployment practices, Docker/K8s compatibility, user control
   - Impact: All configs accessible via `TASK_TIMEOUT`, `PROCESS_KILL_GRACE_PERIOD_MS`, etc.

4. **Pure Event-Driven Architecture (Existing)**
   - Why: Loose coupling, testability, extensibility
   - Impact: All operations go through EventBus, no direct method calls between services

- **Language**: TypeScript with strict type checking
- **Framework**: Node.js with MCP (Model Context Protocol) server
- **Database**: SQLite with better-sqlite3 and WAL mode
- **Key Dependencies**: Zod (validation), simple-git (worktree management), better-sqlite3 (persistence)

### Design Patterns in Use
- **Dependency Injection**: Configuration, Logger, EventBus, all services
- **Repository Pattern**: TaskRepository, OutputRepository abstractions
- **Event-Driven Architecture**: EventBus with handlers for all operations
