## Architecture & Design

### Key Architectural Decisions This Session

1. **Transitive Query Caching Strategy**
   - Two separate caches: `dependenciesCache` and `dependentsCache`
   - Invalidation uses `collectTransitiveNodes()` helper (doesn't use caches to avoid recursion)
   - Cache cleared for task AND all transitively affected nodes

2. **Parallel Validation Safety**
   - `wouldCreateCycle()` creates deep copy of graph (immutable check)
   - Multiple concurrent cycle checks are safe (read-only operations)
   - First failure short-circuits via `Promise.all` semantics

3. **Helper Utilities Pattern**
   - `operationErrorHandler` returns a function suitable for `tryCatchAsync`
   - `emitEvent` encapsulates emission + error logging in one call
   - Both reduce cognitive load and ensure consistent patterns
